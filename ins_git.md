# **Инструкция по работе с Git**

![Эмблема Гит](git.JPEG)

## Предварительная настройка Git

Чтобы "представить" программе git нужно ввести команды:

    git config --global user.name "Ваше имя"
    git config -- user.email "Ваш е-майл"

## Создание репозитория

Чтобы инициализировать новый репозиторий нужно ввести команду:

    git init

## Добавление в индекс

Чтобы добавить изменения в индекс (для фиксации следуещем коммите) нужно ввести комунду:

    git add <имя файла>

## Отображение состояния файлов

Для просмотра состояния файлов нужно ввести команду:

    git status 

## Фиксация изменений

Чтобы зафиксировать изменения, добавленнные в индекс с переходом в текстовый редактор для ввода комментария, нужно ввести команду:

    git commit     

Чтобы зафиксировать изменения, добавленнные в индекс с комментарием без перехода в текстовый редактор, нужно ввести команду:

    git commit -m "message"

Чтобы автоматически индексировать и зафиксировать изменения, добавленнные в индекс, нужно ввести команду:

    git commit -a    

Чтобы автоматически индексировать и зафиксировать изменения, добавленнные в индекс c комментарием без перехода в текстовый редактор, нужно ввести команду:

    git commit -am "message"

## Просмотр истории о созданных коммитах (зафиксированных изменениях) 

Чтобы получить ифнормацию о созданых ранее коммитах нужно ввести команду:

    git log    

Чтобы получить краткую ифнормацию созданых ранее коммитах нужно ввести команду:

    git log --oneline


Чтобы получить ифнормацию о всех созданых ранее коммитах нужно ввести команду:

    git log --all


Чтобы получить краткую ифнормацию о всех созданых ранее коммитах нужно ввести команду:

    git lod --oneline --all 

## Просмотр изменений, сравнение

Чтобы просмотреть текущие изменения нужно ввести команду:

    git diff


Чтобы просмотреть и сравнить изменения между двумя выбранными комитами нужно ввести команду:

    git diff 'hash1' 'hash2' 


## Переключение между коммитами

Чтобы переключится между коммитами с указанным хешем нужно ввести команду:

    git checkout 'hash' 

Чтобы переключится на коммит, на который указывает marter нужно ввести команду:

    git checkout master

# Ветвления

Ветвление стало неотъемлемой частью командной разработки, потому что оно дает возможность работать над разными версиями исходного кода. Основной идеей ветвления является отклонение от основного кода и продолжение работы независимо от него. Также это удобно в тестировании отдельного функционала, потому что позволяет работать над новой частью кода, не беспокоясь о поломке чего-то в рабочей версии. В этой инструкции расскажем о том, как работать с ветками в Git

Имя основной ветки Git-проекта по умолчанию — master (однако зачастую бывает main, например, в GitHub), она появляется сразу при инициализации репозитория.

## Создание новой ветки

Чтобы создать новую ветку нужно ввест комунду:

    git branch <newbranch>

где \<newbranch> это имя создаваемой ветки.

## Перемещение между ветками

Для того чтобы переместиться между ветками необходимо ввести команду:

    git checkout <name of branch>

где \<name of branch> - это название ветки на которую мы перейдем

## Показать удаленные ветки

Для того чтобы вывести только удаленные ветки нужно ввести команду:

    git branch -r

где -r от remote - удаленный 

Для того чтобы ввести локальные и удаленные ветки нужно ввести команду:

    git branch -a

где -a от all - все

##  Создание ветки и перемещение в созданную ветку

Для того чтобы одной командой создать ветку и переместиться на созданную ветку, нужно ввести команду:

    git checkout branch <name of branch>


or

    git checkout -b <name of branch>

## Слияние веток

Слияние — перенос изменений с ветки <name of branch> в текущую ветку

Для слияние веток нужно ввести команду:

    git merge <name of branch>

## Конфликт

Слияние веток может породить кофликт. 
Он вызвано тем, что изменения удаляют или переписывают информацию в существующих файлах. При попытке некорректного слияния Git останавливает выполнение команды, чтобы мы могли разрешить конфликт. 

Решить конфликт можно двумя способами:

1) Вручную разрешить файловый конфликт. Для этого нужно самим изменить файлы, с которыми возникли проблемы. Мы получим файлы такими, какими и представляли их при попытке слияния.

2) Выбрать более подходящий файл, а от второго отказаться.

### Прервать слияние 

Для того чтобы прервать слияние и вернуть все к началу нужно ввести команду:

    git merge -- abort

### Продолжить слияние после разрешения конфликта

Для того чтобы продолжить слияние после разрешения конфликта нужно ввести команду:

    git merge -- continue

## Просмотр последнего коммита на каждой из веток

Чтобы посмотреть последний коммит на каждой из веток, нужно выполнить команду:

    git branch -v

## Просмотр списка слитых веток 

Для того чтобы получить список веток которые уже были слиты нужно ввести команду:

    git branch --merged
